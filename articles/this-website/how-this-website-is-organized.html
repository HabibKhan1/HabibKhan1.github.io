How This Website is Organized

One may wonder, how is this website organized? Well, let's look at its GitHub repository!

Taking a look at this, we see several files/directories, along with a README, a website logo, some JavaScript, CSS, and a home page (in HTML, of course.)

The index.html file constitutes the HTML for the home page. Every folder has one of these. Basically, if you refer your browser to one of these folders (like, for instance, my about page), it will use the index.html file by default to display what will be on your screen.

Sometimes, folders have more than one HTML file. Why is this the case? Well, sometimes, as is the case for the "articles" folder, there are multiple articles, of course. So essentially, grouping HTML files in folders is purely for convenience.

The "img" folder, on the other hand, is only for storing screenshots and images, like the ones shown.

So, that's basically how this website is organized! Hopefully, this clarifies the purpose of having so many folders in my repository.
